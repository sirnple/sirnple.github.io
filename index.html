抛开IDE，重新学习javac

发表于 2020-04-12

抛开IDE，重新学习javac

写这篇博客的原因

在我之前的开发中，编写程序后，通过IDEA的maven插件就打包成jar，但是经常在运行jar时，报出找不到主类的错误，所以就此想爆开IDE，来看看原生的javac的使用，看看能否解决这个问题

前置概念

在说javac之前，我们先来复习一下三个概念，classpath和jar包和package

  1. classpath
    jvm在运行程序时，会去classpath下寻找需要加载的类（除了系统类，系统类的加载不通过classpath），如果你运行程序时，需要将用到的jar包或者类添加到classpath下
  2. jar包
    jar包本质就是一个zip包，将多个类合成一个文件，便于传输和使用
  3. package
    包是java组织类的一种机制，一个类的唯一标识就是它的全类名（QualifiedName），即包名.类名

javac是如何运行的

现在我抛去IDEA的Java项目的结构，直接在任意一个文件夹下写了一个类MainApp.java（运行时打印出hello world）

位于则通过

javac MainApp.java

就可以编译出MainApp.class，此时MainApp.java文件和MainApp.class文件在同一个目录下，执行

java MainApp

就可以打印出hello world
